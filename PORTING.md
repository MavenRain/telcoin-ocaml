# Porting map: telcoin-network (Rust) → telcoin-ocaml

This file is the re-sync contract. Each row maps a Rust source area to the
OCaml module that models it, so the OCaml can be diffed against upstream as the
Rust node moves. The port is **protocol-first**, not file-for-file: the OCaml is
organised around the DAG-consensus protocol itself, with a pure functional core
and an imperative shell, so one OCaml module may correspond to several Rust
files and vice versa.

Upstream reference: `~/Documents/telcoin-network`, a Narwhal + Bullshark DAG
consensus layer over a reth (Ethereum) execution client, tag context around
`v1.11.3` reth deps.

## Status legend

- **done** — implemented and tested in this milestone
- **partial** — modelled, with documented simplifications
- **planned** — signature/plan exists, implementation pending

## Consensus (`crates/consensus`) — parts 1, 2 and 3 done

| OCaml module | Rust source | Status | Notes |
|---|---|---|---|
| `Tn_consensus.Vote_aggregator` | `aggregators/votes.rs` (`VotesAggregator`) | done | Claims an author's slot on first sight (before validating, so a rejected vote burns the author for this header, matching Rust's `authorities_seen`), then validates (wrong header, non-member, bad signature); only accepted votes count toward quorum. `add` always returns the advanced state plus a result, and certifies via `Certificate.assemble` the moment accepted signers reach 2f+1. Fresh per proposal. Errors reported in `Certificate.error`. |
| `Tn_consensus.Parent_aggregator` | `aggregators/certificates.rs` (`CertificatesAggregator`) | done | Per-round parent accumulator. A `seen` author set gives equivocation protection and a **weight that never resets** (only added to); a `pending` buffer holds certificates since the last release and is drained on each release, so each release carries the delta, matching Rust's `drain(..)`. Releases at 2f+1 and re-releases on every later straggler; a post-quorum duplicate author releases nothing. The proposer appends deltas, so a cumulative re-release would double-count parent stake there. |
| `Tn_consensus.Dag` | `consensus/state.rs` (`ConsensusState` DAG half) | done | `Round.Map` of `Authority_id.Map` of `Certificate.t` plus a `Header_digest`-keyed secondary index. | `Tn_consensus.Reputation_scores` | `tn-types` `reputation.rs` (`ReputationScores`) | done | Per-authority leader-support tallies for one schedule window plus a `final_of_schedule` flag. Rust's two asserts (full committee coverage; `total_authorities == size`) become smart-constructor invariants: `fresh` seeds every member, `bump` (`Map.update … (Option.map succ)`) can only touch existing keys. `by_score_desc` breaks ties by **descending** id, fixing the swap-table carve order. |
| `Tn_consensus.Leader_schedule` | `LeaderSchedule`/`LeaderSwapTable` | done | Round-robin `(r/2 - 1) mod size` over the id-sorted committee, plus a reputation-derived swap table (bad = low scorers within a descending std-ceiling capped at `size * pct / 100`; good = high scorers). Three Rust panics vanish into types (odd-round `Leader_round.t`, non-empty good list, `Threshold.of_percent ∈ [0,33]`). **Documented divergence:** the swap RNG is the house SplitMix64 `Prng` seeded by the queried round, not Rust's ChaCha12; determinism and the good/bad sets are identical, the concrete pick differs only when >1 good node exists — deferred behind the `Prng` seam. |
| `Tn_consensus.Sub_dag` | `CommittedSubDag` (`primary/output.rs`) | done | One commit's flattened certificates as bare headers (ascending round, leader last, so `Nonempty.last` is total), running scores, monotone commit timestamp (raw `stored` for the digest, zero-fallback getter for the view — kept as two functions), and signature-derived randomness. `preimage : t -> string` is factored out and documented as the frozen wire-compat byte layout (header digests ++ BCS scores ++ 8-byte-LE timestamp ++ randomness); `digest` domain-tags and hashes it. |
| `Tn_consensus.Bullshark` | `bullshark.rs` (`process_certificate`, `commit_leader`, `order_leaders`, `linked`) + `utils.rs` (`order_dag`) | done | The commit rule. `outcome = Committed of Sub_dag.t Nonempty.t \| No_commit of reason` makes Rust's commit-with-empty-list override unrepresentable; `Schedule_changed` is an internal bounded recursion that never escapes. Replays the Rust `bullshark_tests` scenarios (commit-one, round-robin, missing/dead leaders, weak/not-enough support, GC, reputation-reset cadence, and both schedule-change cases incl. the singleton-good swap) as alcotest cases. Two upstream-test expected multisets were corrected where they assumed a watermark of 6 for round-6 siblings that are only committed at 5. |
| `Tn_consensus.Proposer` | `proposer.rs` (`Proposer` task) | done | Pure Mealy machine `step : t -> now -> input -> t * action list`. Timer **generation counters** kill stale re-arms (critic HIGH): every re-arm bumps a monotone `gen`, a `Timer_fired` with a stale `gen` is discarded — the pure stand-in for `Interval::reset`. Propose condition `enough_parents && (max_timed_out \|\| (advance_round && (enough_digests \|\| min_timed_out)))`; parents jump/extend/ignore by round comparison; sticky timeout flags cleared only on a proposal; `process_committed_headers` digest re-queue (skipped headers' payloads prepended, oldest-round-first). **Timing chunk (ported):** the proposer now holds the `Leader_schedule` (refreshed by the Node after each commit, the pure stand-in for Rust's shared `Arc<RwLock<_>>`). *Leader fast path* — proposing the round before its own even leader round halves the max header delay and collapses the min to zero (`calc_max_delay`/`calc_min_delay`), chosen at re-arm time under the current generation. *Readiness gate* — the `advance_round` conjunct withholds an early (pre-max-timeout) proposal until `ready` holds (even round: the round's leader certificate is in the held parents via `update_leader`; odd round: the leader votes have settled via `enough_votes`, f+1 for it or a 2f+1 quorum against); the max deadline overrides it, so liveness is unchanged. The gate is recomputed from `ready` at the end of `apply_parents` and cleared on every proposal, mirroring Rust exactly. **Still deferred:** the equivocation-guard re-emit of a stored header is present for the recovery chunk but unreachable in forward operation, so `step` needs no error result. |
| `Tn_consensus.Voter` | `certifier.rs` + `network/handler.rs` (`RequestHandler::vote`) | done | Pure `vote : t -> dag -> now -> Header.t -> t * decision`. **Vote-once** keyed by author (latest vote): identical re-request → `Recast` the stored vote, a different header for a voted round → `Reject Equivocating_header`, a lower round → `Reject Already_voted_higher`. Parents resolved genesis-first then the DAG (the round-1 genesis-parent rule falls out), checked for one-round-below layering, `2f+1` quorum over distinct origins, and `created_at` monotonicity (inclusive, equality allowed). A Byzantine peer is protocol-normal, so **every outcome is a `decision`, never an error** — the genuine invariant break (two conflicting *certificates*) is the DAG's, surfaced at the Node. **Timing chunk (ported):** the future-timestamp check now applies Rust's `max_header_time_drift_tolerance` (whole seconds, default 1): a header stamped up to one second ahead of `now` is voted (Rust waits out the small drift then votes; the pure equivalent is an immediate accept), beyond that is `Reject Future_timestamp`. Slice-deferred (IO): batch sync, evaluation timeout, execution-result waits — an unresolved parent yields `Need_parents` and the peer re-requests. |
| `Tn_consensus.Node` | `primary.rs` + `consensus/mod.rs` | done | Composes Proposer + Voter + `Vote_aggregator` + per-round `Parent_aggregator`s + `Bullshark` into `step : t -> now -> event -> (t * command list, error) result`. Every intra-node channel collapses into direct composition; only true message crossings are events/commands. **`error` is reserved ONLY for the three DAG invariant breaks** (certificate equivocation, missing parent / parent-round) (critic HIGH): wrong-epoch and below-GC certificates are `Ok` no-ops, invalid/duplicate/superseded votes and equivocating peer *headers* produce a command or nothing. A formed certificate is self-inserted through the same spine as gossip and broadcast; released parent quorums drive the next proposal; committed sub-DAGs are emitted in order and prune the proposer + GC the aggregators. |

## Shell + execution seam + state foundation — done; network, storage — planned

| OCaml module | Rust source | Status | Notes |
|---|---|---|---|
| `Tn_sim.Sim` | tokio runtime + channels (port-only) | done | The deterministic discrete-event simulator that closes the pure core's loop — the runnable vertical slice. Holds one `Node.t` per authority (`Authority_id.Map`), a revealed-certificate set (`Map` keyed by header digest), and a time-ordered event queue (`Map` over `(delivery_ms, seq)`; the monotone `seq` makes the key a total, collision-free order so simultaneous events replay deterministically). `run` is a tail-recursive fold: pop the earliest event, advance the clock, `Node.step`, then interpret each command into future scheduled events — `Broadcast_header` → `Vote_request` to each peer with parents resolved from the revealed set; `Send_vote` → `Vote_received` unicast; `Broadcast_certificate` → reveal + `Certificate_received` to peers; `Arm_timer` → self `Timer_fired` after the span; `Send_missing_parents` answered from the revealed set; `Emit_committed` → the node's output log. The only nondeterminism is per-message latency from a single seeded `Prng`. Pure library: no imperative loops, exhaustive command match, `Option.fold`/`Result.fold` throughout. Observability: `committed`/`commit_count`, `error` (the halting invariant break, if any), `agreement` (pairwise prefix-consistency of the committed logs — a `Diverge` is a safety violation). **Documented slice model:** the clock is milliseconds coarsened to whole seconds for the digest-domain `Timestamp` (a delivery is never earlier than its send, so a voter's clock never trails a header's `created_at`); the shared revealed-certificate set stands in for the fetch protocol a later network chunk adds; honest, fully-connected, reliable delivery only (no Byzantine/loss model yet). |
| `bin/tn_sim` | `bin/telcoin-network` | done | Stands up a 4-validator honest committee (seed 42, 5–50 ms latency, 20 s horizon), runs `Sim` to the horizon, and prints per-node commit counts, the agreement verdict, and node 0's committed leader sequence; exits non-zero on any invariant break, disagreement, or empty output, so it doubles as a smoke test. Default run: all four nodes commit an identical 23-sub-DAG prefix following the round-robin schedule (round 2 → authority 0, 4 → authority 1, …); the leader fast path lets each anticipated leader propose sooner, so more rounds commit within the same horizon than the pre-timing-chunk slice (19). Optional flags `--validators N`, `--seed S`, `--until-s T` (each total, clamped/defaulted) override the run; the latency band lives on `Sim.config`. |
| `Tn_execution.Consensus_block` | `ConsensusHeader` (`types/src/primary/block.rs`) | done | The consensus-chain block: `parent_hash`, one committed `Sub_dag`, a monotone `Number` (ancestor count; genesis anchor 0, first committed block 1), and a cached `digest`. `preimage` is the frozen wire-compat layout — `parent_hash` ++ `sub_dag.digest` ++ 8-byte-LE `number` ++ 32 zero bytes (Rust's unused `extra` `B256::default()`) — exactly the field order of `ConsensusHeader::digest_from_parts`; `digest` domain-tags (`tn:output`) and hashes it. **Genesis-anchor divergence:** Rust's first block links to `ConsensusHeader::default().digest()` (a header wrapping a default certificate); reproducing that needs a `Certificate::default` the port's types forbid (a certificate is proof of a quorum), so `genesis_parent` is a distinct documented constant, byte-compat-deferred with the hash swap. |
| `Tn_execution.Engine` (`ENGINE` / `Noop`) | `executor/src/subscriber.rs` (the output→header fold) | done | The execution seam. `module type ENGINE` is `execute : t -> Sub_dag.t -> (t * block, error) result` + `tip`/`height`. `Noop` folds committed sub-DAGs into the `Consensus_block` chain exactly as the Rust subscriber does (`number = prev + 1`, `parent_hash = prev.digest`) and **cannot fail** — its `error` is the uninhabited `Nothing.t` (OCaml's `Void`/Rust's `!`), so `execute` is total and the impossible error branch is discharged by `Nothing.absurd`. The real OCaml EVM links here later with a populated `error`. |
| `Tn_sim.Sim` (execution view) | `executor` subscriber wiring | done | The shell derives each node's chain on demand: `executed` / `execution_tip` fold `Noop` over `committed authority`. Pure and outside the event loop, so the honest run stays byte-for-byte deterministic; `bin/tn_sim` prints node 0's chain height and tip. |
| `Tn_state.U256` | `alloy_primitives::U256` (`ruint::Uint<256, 4>`) | done | The 256-bit EVM word, held as its canonical 32-byte big-endian encoding so `String.compare` is the unsigned order and the byte codec (`to_be_bytes`/`of_be_bytes`) is the identity — matching `U256::to_be_bytes` / `U256::from_be_slice`. Wrapping and checked `add`/`sub` via an LSB→MSB carry/borrow fold; `of_int`/`to_int`, hex codec. Multiplication (the low half of a full 512-bit schoolbook product), unsigned division and remainder (binary long division, tracking the bit shifted out of the top so a shifted remainder past `2^256` still compares `>=` the divisor), modular add and multiply reduced from the true 257- and 512-bit intermediates, exponentiation, and the bitwise and shift operations arrived with the EVM ALU chunk. The module stays **mathematically honest**: division and remainder by zero are `None`, never a silently defined value. The EVM's *total* semantics, which map those to zero, live in `Tn_evm.Alu`. `to_int` is the partial inverse of `of_int` the interpreter converts offsets, lengths and jump destinations with. |
| `Tn_state.Storage` | `revm-state` `EvmStorage` (`Address → StorageSlot`) | done | One account's storage: an infinite array of zero words of which a finite set has been written, so `get` is total and the module has no error type. **Canonical:** writing zero *removes* the key rather than storing it, so `equal` is exact content equality, `is_empty` is a constant-time map test rather than a scan, and a contract that clears every slot it wrote leaves a storage `equal` to the one it started from. The storage root (the trie hash committing the map) needs the state trie and RLP, which the port still defers; keccak itself now exists in `Tn_keccak`. |
| `Tn_state.Address_word` | `revm-interpreter/src/instructions/utility.rs` (`Address::from_word` / `into_word`) | done | The two total conversions between a 20-byte address and a 256-bit word. `to_word` left-pads with twelve zero bytes and is injective; `of_word` keeps the low twenty bytes and **discards the high twelve without complaint** — a word with dirt above bit 160 is not an error for `BALANCE`, it is a query for the truncated address, so hardening `of_word` into a partial function would halt valid mainnet transactions. It lives in `tn_state` because it must see both `Units.Address` and `U256`, and neither owns the other. |
| `Tn_state.Account` (+ `Nonce`) | account state (`nonce: u64`, `balance: U256`, storage) | done | An account: `Nonce` (saturating u64 counter), `balance` (`U256`), and — since the host seam — its `Storage.t`, with checked `credit`/`debit`, `increment_nonce`, `slot`/`set_slot`, and **two distinct emptiness predicates**. `is_empty` is the literal EIP-161 test (zero nonce, zero balance and no code) — storage is deliberately *not* a conjunct, because EIP-161 does not make it one; it is what state-clearing, the EIP-1052 `EXTCODEHASH` fold-to-zero and the `CALL`-to-empty surcharge use. `is_absent` is `is_empty && Storage.is_empty`, and is what the world state prunes on. `storage_root` needs the state trie and RLP, which the port still defers. Code now lives on the account (`Bytecode.t`, set through `with_code`), so `code_hash` is `Bytecode.hash` — Keccak of the bytes, `KECCAK_EMPTY` for empty code — and `is_empty`'s third conjunct is live. A run itself still deploys no code (that is `CREATE`), so contracts reach the state at genesis or through `set_account`. |
| `Tn_state.World_state` | the execution state map (`Address → Account`) | done | An address-keyed `Map`, total on addresses (an absent address reads `Account.empty`) and total again at the slot level (`storage` of a slot of a nonexistent account is zero). Storage lives *inside* the account entry rather than in a second address-and-slot map, so orphan storage is not representable and `remove_account` cannot forget half its job. Kept canonical so `equal` is exact content equality and two nodes that applied the same transactions from the same genesis hold identical states (the state-level corollary of consensus agreement). **The pruning predicate is `Account.is_absent`, not `Account.is_empty`.** The distinction is load-bearing: `is_empty` ignores storage (correctly — EIP-161 ignores it), so pruning on it would erase an `SSTORE` into a zero-nonce zero-balance account and `equal` would stop being exact, since two states differing in that slot would compare equal. `is_absent` is strictly stronger and covers every field `Account.equal` compares, which is exactly what canonicity needs; on a real chain the two agree on every account that can exist (reaching storage needs `SSTORE`, which needs code, which falsifies `is_empty`), and this port errs toward keeping the account. `of_alloc` is the genesis funding (`GenesisAccount::with_balance`) and is the one constructor that has fallen behind `Account.t` — it cannot express pre-populated genesis storage. A trie state root is deferred with the rest of the crypto. |
| `Tn_state.Transfer` | value transfer (revm `transfer` + `deduct_caller`) | done | A value-transfer transaction plus its pure state transition `apply : World_state → t → (World_state, error) result`, the first engine behind the `ENGINE` seam to carry a real `error` (`Nonce_mismatch` / `Insufficient_balance` / `Balance_overflow`) in place of `Noop`'s uninhabited one. Exact-nonce and balance checks; the recipient is credited from the already-updated state so a self-transfer nets to zero and still advances the nonce; any error reverts atomically. **Deferred inclusion divergence:** `Nonce_mismatch` and `Insufficient_balance` are reth's pre-execution rejections (no state change — faithful); `Balance_overflow` is a mid-execution failure that reth includes as a failed tx (nonce advanced, gas charged, value reverted). This gasless layer reverts it atomically; the included-and-failed nonce advance + fee charge belong with the gas / block-execution chunk. Not yet wired into the running slice (awaits batch payloads, networking-deferred). |
| `Tn_evm.Alu` | `revm-interpreter/src/instructions/{arithmetic,bitwise}.rs` | done | The **total** EVM semantics of every arithmetic, comparison, bitwise and shift opcode, over the honest (partial-on-zero-divisor) `U256` arithmetic: `DIV`/`MOD` by zero are zero, `SDIV`/`SMOD` read two's complement (divide magnitudes, re-sign by the sign exclusive-or — the lone `-2^255 / -1` overflow falls out unforced), `ADDMOD`/`MULMOD` reduce the true 257-/512-bit intermediate, `SIGNEXTEND`, the one-or-zero comparison words, `BYTE`, and `SHL`/`SHR`/`SAR` with their 256-bit boundaries. Pure functions of their word operands — no gas, no stack, no state. Held to an independent zarith oracle by 25 property tests. |
| `Tn_evm.Opcode` (+ `Depth`) | `revm-bytecode/src/opcode.rs` (the `opcodes!` table) | done | The instruction set as a sum type, with `decode`/`to_byte` round-tripping over all 256 bytes and `immediate_bytes` for the `PUSH` widths. Every operand a byte can carry is a smart-constructed newtype — `Push_bytes` (1–32) and `Depth` (1–16) — so `PUSH33` or `DUP17` is unrepresentable rather than rejected at run time. **Scope divergence:** the *single-frame* subset is defined — arithmetic, comparison, bitwise and shift (dispatched to `Alu`), stack, memory, gas and control flow, plus, since the host seam, the block and transaction context (`COINBASE`…`BASEFEE`, `0x41`–`0x48`; `ORIGIN`/`GASPRICE`), the frame's own context and calldata and code (`ADDRESS`…`GASPRICE`, `0x30`–`0x3a`), account storage (`SLOAD` `0x54`, `SSTORE` `0x55`) and `MCOPY` (`0x5e`). Since the hash landed it also covers `KECCAK256` (`0x20`), the logs (`LOG0`-`LOG4`, `0xa0`-`0xa4`, one constructor carrying a `Topic_count.t`) and EIP-1153 transient storage (`TLOAD` `0x5c`, `TSTORE` `0x5d`); and, once code landed on an account, the external-code readers (`EXTCODESIZE` `0x3b`, `EXTCODECOPY` `0x3c`, `EXTCODEHASH` `0x3f`, the last carrying EIP-1052's fold-to-zero). What remains absent is what needs a **second frame** or a piece of state the port has not built: `RETURNDATASIZE`/`RETURNDATACOPY`, `BLOCKHASH`, the blob instructions, the calls, the creations and `SELFDESTRUCT`. Each still decodes to `None` exactly as an unassigned byte does, so a program using one halts rather than silently doing something else — being absent from a total decoder *is* the mechanism, needing no allow-list and no feature flag. `PREVRANDAO` is named for the post-merge reading of `0x44` rather than revm's retained `DIFFICULTY`, because this port targets Prague, where the pre-merge reading is not representable. |
| `Tn_evm.Stack` | `revm-interpreter/src/interpreter/stack.rs` | done | The 1024-word operand stack, immutable and all-or-nothing: the limit is checked *before* a push, and a multi-word pop either takes every word or none. **Deliberate divergence:** revm reports a `DUP`/`SWAP` *underflow* as `StackOverflow` (its stack helper returns a bare bool the instruction wrapper cannot tell apart, `instructions/stack.rs:44-62`); this port reports the condition that actually held. Nothing observable turns on it — every exceptional halt discards the frame, consumes all gas and returns no output, so the reason is diagnostic only. |
| `Tn_evm.Memory` | `revm-interpreter/src/interpreter/shared_memory.rs` | done | Byte-addressed frame-local memory: reads are total (an unwritten byte is zero), expansion is `O(1)` and rounds up to whole 32-byte words, and only nonzero bytes are stored so `equal` is exact content equality. Charging is deliberately *not* here — `words_needed` reports what an access requires and the interpreter pays for the difference, since only it holds the gas. A zero-length access needs no words at any offset, which is the rule `RETURN`/`REVERT` rely on. The host seam added `store_bytes`, the bulk writer for `CALLDATACOPY`/`CODECOPY`/`MCOPY`: it folds `store_byte`, so the canonicity rule that a zero byte *removes* its key is stated once — a copy length routinely exceeds its source length, so writing long runs of zeroes is the common case, not a corner. `MCOPY` needs no temporary buffer and no direction test, because `slice` reads the *old* memory and `store_bytes` writes a new one; revm must reason about overlap direction in a flat buffer (`instructions/memory.rs:81`), a persistent map does not. |
| `Tn_evm.Gas` | `revm-context-interface/src/cfg/gas.rs` + `revm-interpreter/src/instructions.rs` (the static table) | done | The Cancun/Prague schedule: the named tiers (`ZERO` 0, `BASE` 2, `VERYLOW` 3, `LOW` 5, `MID` 8, `HIGH` 10, `JUMPDEST` 1) as a per-opcode static cost charged *before* the instruction body, `EXP`'s 50-per-exponent-byte dynamic cost, and the memory curve `3w + w²/512` charged as the difference against what the frame already paid. | `Tn_evm.Effects` | `revm-context-interface/src/journaled_state.rs` (`Journal`) + `journaled_state/entry.rs` (`JournalEntry`) | done | Everything a frame can change outside itself, as one persistent value: the world state, the `Access.t` and the `Refund.t`. **This is the module where revm's mutable undo journal has no counterpart.** revm needs one because its state is a mutable map with no older version left to return to; here all three components are persistent, so the value that *was* the state is still in the caller's hand.
